🍞 BREAD · The stack

Four layers, one direction, and the app that runs them.

BREAD is a no-build, server-rendered hypermedia stack, built in four layers that each stand only on the ones below: BATCH, GRAIN, MILL, PROOF. PANTRY is not a fifth layer in that chain: it's the app that composes all four into one running server, the same job this site does. This whole site runs on the same four layers. Start here, then open any member from the cards below.

The layers
🥖 BATCHSubstrate
Live · runs this site

The bottom layer: server-rendered hypermedia on Bun with no build step and near-zero client JavaScript. Bun runs the TypeScript directly, nothing between source and server.

See the substrate →
🌾 GRAINDesign system
Live

On top of BATCH: an AI-interaction design system where a person and an AI operate the same controls through one vocabulary, with the AI's presence shown as a visible signal (grain = AI).

See the system →
⚙ MILLContent engine
Live · renders the notes and layer docs

Above both: a Markdown-to-pages engine that renders GRAIN pages from content. It manages this site's notes and docs, a layer above BATCH and GRAIN, an extension of neither.

See the engine →
📋 PROOFAI plan board
Live · core, board, check/init, and the live SSE board all shipped

Above MILL: plans are markdown files, and PROOF's board is a projection of them, never the other way round. The AI maintains the files; the board is for the human watching.

See the board →
The app that runs them
🥫 PANTRYDev-docs cockpit
Live · the plan board as home, AI-retrieval (/llms.txt, /knowledge.json), the /map mindmap, ⌘K search, plus docs, reference, catalog, and standards

PANTRY is an installable server, AI-legible not AI-powered (it runs no model), that composes BATCH, GRAIN, MILL, and PROOF into one project's docs and plan board. Drop it into any repo; it's the neutral, project-agnostic sibling of this site.

See the cockpit →

This portfolio is the other consumer of the same four layers: two composition roots, one stack. PANTRY is the neutral, installable one; this site is the personal one, with its own résumé and notes on top.

One direction of dependency

Each layer builds only on the ones beneath it: BATCH imports nothing inward, GRAIN imports nothing from BATCH but one port, MILL sits above both, and PROOF sits above MILL. That purity is what lets each become its own repo once proven.

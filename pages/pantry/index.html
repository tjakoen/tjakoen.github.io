🥫 PANTRY · Dev-docs cockpit

One installable server for the whole stack.

PANTRY is the BREAD stack's installable developer-docs and AI cockpit: one app that composes the layers (BATCH, GRAIN, MILL, PROOF) into a single server you drop into any project. It is an app, not a fifth layer: the neutral, project-agnostic sibling of this portfolio. Two composition roots wiring the same stack is a stronger reuse proof than one.

v2 shipped · home, the board, the framework docs, reference, catalog, and standards, plus an install kit
The idea

An app, not a layer

PANTRY doesn't sit in the batch → grain → mill → proof chain. It mounts all four, the same job this portfolio does, just without the personal branding.

What runs today

A home page, this project's own PROOF plan board, the framework docs rendered through MILL, plus /reference, /catalog, and /standards — all from the installed packages, never copied.

What's next

Package-resolving the assets and layer PLANs at the split, plus the mindmap and AI-retrieval endpoints.

Drop it in anywhere

Verified installed against a scratch external project: bunx it into a repo and it reads that project's own plans/ folder.

The design

The layer-versus-app split, what PANTRY mounts, and the reference/catalog/standards work are written up in the PANTRY plan that ships beside the code.
